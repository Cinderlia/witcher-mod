## 目标

为 Witcher 增加反射型 XSS 识别能力。模块在不改变 AFL 变异核心的前提下，围绕 AFL 产生的 seed 做二次验证与上下文分析，输出高可信的反射型 XSS 证据与复现脚本。

## 总体流程

1. 读取 AFL 产生的 seed
2. 解析出参数键值对
3. 逐个参数替换为随机 payload，生成新 seed
4. 对新 seed 执行一次并收集响应
5. 在响应中检测随机 payload 的反射位置
6. 基于反射上下文构造攻击性 payload，再次执行
7. 对攻击性 payload 的响应上下文进行模板匹配
8. 评估是否可能突破限制并执行，判定为 XSS
9. 保存证据、seed、执行脚本

## 架构分层

### 入口层

- CLI：命令行入口与参数解析
- Pipeline：编排整体流程，串联解析、变异、执行、检测与存储

### 领域层

- SeedParser：解析 seed 为结构化对象
- ParamExtractor：从 GET/POST/COOKIE 中抽取键值对
- PayloadFactory：生成随机 payload 与攻击性 payload
- SeedMutator：按“单键替换”策略生成衍生 seed
- ReflectionDetector：检测响应中随机 payload 的反射位置
- ContextAnalyzer：提取反射上下文并分类（HTML、属性、脚本、URL 等）
- RiskEvaluator：根据模板与上下文评估可执行风险

### 基础设施层

- SeedExecutor：执行单次请求并采集响应
- Storage：持久化 seed、响应片段、上下文与执行脚本
- Clock/Random：可替换的时间与随机数提供器

## 模块接口

### Pipeline

- run(seed_iterable) -> findings
- run_on_seed(seed_bytes) -> findings

### SeedExecutor

- execute(seed_bytes) -> ExecutionResult

### SeedParser

- parse(seed_bytes) -> SeedInput
- extract_params(seed_input) -> List[Param]

### PayloadFactory

- random_payloads(param, count) -> List[Payload]
- attack_payloads(context) -> List[Payload]

### ReflectionDetector

- find_reflections(response_text, token) -> List[ReflectionFinding]

### ContextAnalyzer

- build_context(response_text, positions) -> List[ContextSnippet]
- classify_context(context_snippet) -> ContextType

### RiskEvaluator

- evaluate(context_snippet, payload) -> RiskDecision

### Storage

- save_seed(seed_bytes, metadata) -> path
- save_response(response_text, metadata) -> path
- save_finding(finding) -> path

## 关键数据结构

- SeedInput：cookie、query、post、headers 的分段表示
- Param：键值对与来源（GET/POST/COOKIE）
- Payload：随机/攻击性 payload 与 token
- ExecutionResult：响应文本、状态、耗时、错误信息
- ReflectionFinding：反射位置、上下文、token
- XSSFinding：最终判定、证据、关联 seed 与执行脚本

## 与 Witcher 的集成点

- 输入：从 AFL 输出目录读取 seed 或队列
- 执行：复用 httpreqr 或 CGI 模式的单次执行路径
- 环境：继承 Witcher 的登录态、必需参数与头部
- 输出：存储到 Witcher 的结果目录并可被 reporter 读取

## 扩展点

- 自定义 payload 策略
- 更细粒度的上下文分类器
- 更严格的风险评估策略
- 多响应体解析（HTML/JSON/JS）

## 非目标

- 不替代 AFL 的核心变异逻辑
- 不进行长时间 fuzz
- 不进行浏览器渲染与执行验证
